# Лабораторна робота 5: Помилки роботи з пам'яттю в Linux

## Вариант 14 — Фрагментація пам'яті

Демонстрація випадку, коли велика алокація неуспішна, хоча сумарно вільної пам'яті достатньо.

## Теорія

**Фрагментація пам'яті** — це ситуація, коли вільна пам'ять розбита на малі шматки, і хоча сумарно їх достатньо, великий суміжний блок виділити неможливо.

### Типи фрагментації:
1. **Зовнішня фрагментація** — вільні блоки розсіяні між виділеними
2. **Внутрішня фрагментація** — невикористаний простір у виділених блоках

## Компіляція та запуск

```bash
gcc -g fragmentation.c -o fragmentation
./fragmentation
valgrind --leak-check=full ./fragmentation
```

## Схема фрагментації

```
Етап 1: [a:1KB][b:10KB][c:1KB]

Етап 2 (після free(b)): [a:1KB][ВІЛЬНО:10KB][c:1KB]

Етап 3 (після malloc(8KB)): [a:1KB][d:8KB][c:1KB][ВІЛЬНО:2KB]

Етап 4 (після free(a), free(c)): [ВІЛЬНО:1KB][d:8KB][ВІЛЬНО:1KB]
                                    ↑                   ↑
                            Фрагментовано! Не можна виділити 2KB поспіль

Але malloc розумна — вона об'єднує блоки і виділяє!
```

## Висновок

- Фрагментація призводить до неефективного використання пам'яті
- Хоча сумарно вільної пам'яті 11+ KB, великий суміжний блок може бути недоступний
- Сучасні allocator (glibc malloc) намагаються мінімізувати фрагментацію через об'єднання (coalescing) вільних блоків
- Програма не містить помилок і всі блоки коректно звільнені
