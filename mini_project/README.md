# Мініпроєкт

**Тема:** Спільна пам’ять для обміну даними

---

## Мета роботи

Метою роботи є ознайомлення з механізмами міжпроцесної взаємодії (IPC) у операційній системі Linux за допомогою спільної пам'яті POSIX. Також метою є закріплення навичок роботи із системними викликами `shm_open()`, `ftruncate()`, `mmap()`, `munmap()`, `close()` та `shm_unlink()`.

У процесі виконання роботи необхідно реалізувати передачу даних між двома окремими процесами без використання звичайних файлів. Для цього використовується спільна пам'ять, яка дозволяє декільком процесам працювати з однією областю пам'яті.

---

## Опис реалізації

У рамках роботи було реалізовано дві окремі програми мовою програмування C:

- `writer.c` — програма для запису даних у спільну пам'ять;
- `reader.c` — програма для читання даних зі спільної пам'яті.

Для передачі інформації між процесами використовується структура `SharedData`, яка містить такі поля:

- температура пристрою;
- напруга;
- стан пристрою;
- текстове повідомлення.

У програмі `writer.c` за допомогою системного виклику `shm_open()` створюється об'єкт спільної пам'яті. Після цього за допомогою `ftruncate()` задається необхідний розмір області пам'яті. Далі виклик `mmap()` відображає цю область у адресний простір процесу, що дозволяє працювати з нею як зі звичайною структурою даних через вказівник.

Після відображення пам'яті програма записує у структуру тестові дані:

- температуру;
- напругу;
- стан пристрою;
- текстове повідомлення.

Програма `reader.c` відкриває той самий об'єкт спільної пам'яті у режимі читання та також використовує `mmap()` для доступу до пам'яті. Після цього дані зі структури виводяться на екран.

Після завершення роботи обидві програми звільняють ресурси за допомогою `munmap()` та `close()`. Також використовується `shm_unlink()` для видалення об'єкта shared memory із системи.

У програмі реалізована перевірка помилок системних викликів. У разі виникнення помилки використовується функція `perror()`, після чого програма завершує роботу з кодом помилки.

---

## Використані системні виклики

### `shm_open()`

Системний виклик `shm_open()` використовується для створення або відкриття об'єкта спільної пам'яті POSIX. У програмі `writer.c` він створює нову область shared memory, а у `reader.c` відкриває вже існуючу область для читання даних.

### `ftruncate()`

Функція `ftruncate()` використовується для встановлення розміру об'єкта спільної пам'яті. Після створення shared memory її розмір дорівнює 0 байт, тому необхідно явно задати потрібний розмір.

### `mmap()`

Системний виклик `mmap()` використовується для відображення області пам'яті у адресний простір процесу. Після цього процес може працювати зі спільною пам'яттю через звичайний вказівник.

### `munmap()`

Функція `munmap()` звільняє область пам'яті, яка була відображена за допомогою `mmap()`.

### `close()`

Системний виклик `close()` використовується для закриття файлового дескриптора shared memory.

### `shm_unlink()`

Функція `shm_unlink()` видаляє об'єкт shared memory із системи після завершення роботи програм.

### `perror()`

Функція `perror()` використовується для виведення повідомлень про помилки системних викликів.

## Приклад запуску

### Компіляція програм

```bash
gcc writer.c -o writer
gcc reader.c -o reader
```

### Запуск програм

Спочатку необхідно запустити програму запису:

```bash
./writer
```

Після цього запускається програма читання:

```bash
./reader
```

## Висновок

У ході виконання роботи було досліджено механізм міжпроцесної взаємодії у Linux за допомогою спільної пам'яті POSIX. Було реалізовано дві окремі програми для запису та читання структури даних через shared memory.

Під час виконання роботи були використані системні виклики `shm_open()`, `ftruncate()`, `mmap()`, `munmap()`, `close()` та `shm_unlink()`. Також була реалізована перевірка помилок системних викликів, що підвищує надійність роботи програм.

У результаті роботи було отримано практичні навички використання shared memory для обміну даними між процесами в операційній системі Linux.
