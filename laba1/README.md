# Завдання 1.14

Програма `last` читає останні n рядків із файлу або стандартного вводу. Використовується кільцевий буфер, тому весь файл у пам'ять не завантажується.

---

## Основні можливості

- `-n N` — кількість рядків (за замовчуванням 10)
- `-r` — виведення у зворотному порядку
- `-m` — примусове використання mmap
- Автоматичний вибір між `mmap` (файли >1MB) та `fgets` (stdin, малі файли)

---

## Компіляція

```bash
gcc -O2 -Wall -Wextra -o laba1 laba1.c
```

---

## Інструкція з використання

**З файлу (останні 5 рядків):**
```bash
./laba1 -n 5 test.txt
```

**У зворотному порядку:**
```bash
./laba1 -n 5 -r test.txt
```

**З stdin:**
```bash
seq 1 20 | ./laba1 -n 3
```

**Примусовий mmap:**
```bash
./laba1 -n 5 -m test.txt
```

---

## Архітектура та принцип роботи

**Кільцевий буфер**

Масив фіксованого розміру. Коли буфер заповнений, новий рядок перезаписує найстаріший. Змінні `first` та `have` відслідковують стан буфера.

**Читання fgets**

Використовується для stdin і файлів менше 1MB. Потокове читання рядок за рядком.

**Читання mmap**

Для великих файлів файл відображається у пам'ять. Програма йде з кінця до початку, шукаючи `\n`. Це швидше, бо менше системних викликів.
