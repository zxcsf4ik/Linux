# Лабораторна робота 11: Реальні сигнали та timer_create()

## Завдання

Написати програму, яка посилає реальні сигнали за допомогою `timer_create()` та аналізує кількість оброблених і втрачених сигналів.

## Теорія

### Реальні сигнали (Real-time signals)
- Мають діапазон від `SIGRTMIN` до `SIGRTMAX`
- На відміну від звичайних сигналів, вони ставляться у чергу (не втрачаються, якщо надходять одночасно)
- Дозволяють передавати додаткову інформацію разом із сигналом

### timer_create() та timer_settime()
- `timer_create()` — створює інтервальний POSIX-таймер для процесу
- `timer_settime()` — запускає, зупиняє або змінює параметри створеного таймера
- Дозволяють автоматично та регулярно надсилати сигнали через заданий інтервал часу

### sigaction()
- Використовується для встановлення або зміни обробника сигналу
- Прапорець `SA_SIGINFO` дозволяє обробнику отримувати розширену інформацію про сигнал через структуру `siginfo_t`

## Висновки

- Було досліджено та практично освоєно роботу з реальними сигналами у системі Linux
- `timer_create()` дозволяє створювати точні POSIX-таймери для автоматичної генерації подій за розкладом
- На практиці перевірено, що при великій швидкості та інтенсивності надсилання частина сигналів може втрачатися
- Системний виклик `sigaction()` забезпечує надійну обробку сигналів та отримання додаткових даних у процесі
