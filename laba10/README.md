# Лабораторна робота 10: fork() та exec()

## Завдання

Написати дві програми: одна просто виводить повідомлення, інша — викликає її після fork() за допомогою exec().

## Теорія

### fork()
- Створює дочірній процес (копія батька)
- У дочірнього процесу новий PID
- Повертає 0 у дочірньому, PID дочірнього у батькові

### exec()
- Замінює образ поточного процесу новою програмою
- execl() — передає аргументи як список строк
- Повертається лише при помилці

## Компіляція та запуск

```bash
gcc child.c -o child
gcc parent.c -o parent
./parent
```

## Висновки

- fork() дозволяє створювати нові процеси
- exec() замінює образ поточного процесу
- Комбінація fork() + exec() — основа для запуску програм у Unix
- wait() синхронізує батька та дочірнього процесу
